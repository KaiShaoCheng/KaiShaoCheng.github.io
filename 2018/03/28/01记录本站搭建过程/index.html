搭建hexo/配置

这一步折腾了2天，要不是有丰富的文章可以借阅，根本不会成功。
需要注意的是，大部分文章所使用的版本都比较早，需要自行替换成新的版本。

CentOS下搭建Hexo + github 博客 —— MaxZing

· 文章里面的Node.js版本比较早，指令没问题，当前（2018-03-28）最新版本是 node-v8.10.0 把文章里面的wget的地址替换成 nodejs.org/dist/v8.10.0/node-v8.10.0-linux-x64.tar.xz
· 文章里面原作者配置环境的时候使用root用户，安装Hexo的时候使用另一个用户。我并没有分用户操作，全部在root下完成。
· 部署到GitHub步骤中，SSH密钥在服务器上生成，需要全部复制到Github配置页面去，包括邮箱账号。
· 如果使用教程里面的NexT主题，配置方面，直接参考主题原作者的教程更快更准确。

NexT使用文档 —— IIssNan

配置主题显示的语言，需要在服务器上配置Hexo自身的配置文档( Hexo/_config.yml )里面的language，设置为zh-Hans，与NexT主题里面的language文件夹内的预置文件才能吻合。不是在主题的配置文档（ Hexo/themes/next/_config.yml ）里面配置了。

发布第一篇文章

应该就是这篇文章了。
MarkDown编写，在线编辑器或是下载安装编辑器，写好以后保存成.md的文档。
开头格式如下： 

title: xxx
date: YYYY-MM-DD HH:MM:SS
author: xxx
categories:
- 指南
tags:
- Hexo
- ASF
- BBR
---

然后上传到服务器 hexo/source/_posts/ 里面，再执行clean并重新generate和deploy（要在hexo文件夹下执行）。
